# PR flow (shared by every skill that writes to docs/)

Every skill that produces or modifies files under `docs/` finishes by opening a **pull request** against the repo's default branch. This is the canonical deliverable. Local-only writes are a fallback for when credentials aren't available.

## Credentials detection (in order)

1. **`gh` CLI authenticated** (`gh auth status` exits 0) — preferred. Handles auth and PR creation cleanly.
2. **`GITHUB_TOKEN` env var** present — use the token for both push (via `https://<token>@github.com/...` remote) and the GitHub REST API for PR creation.
3. **Tokenized clone URL** (the user passed `https://<token>@github.com/...` to document-init) — reuse that token.
4. **None of the above** — fall back to local-only mode and print explicit setup steps for the user.

## Branch naming

Use `business-docs/<skill>-<short-timestamp>`. Examples:
- `business-docs/init-20260428-1530`
- `business-docs/refresh-20260428-1830`
- `business-docs/module-billing-20260428-1900`

Short timestamp = `YYYYMMDD-HHMM` (UTC). One branch per skill run keeps PRs reviewable independently.

## Steps

```bash
# 1. Create branch from current HEAD
cd <repo>
BRANCH="business-docs/<skill>-$(date -u +%Y%m%d-%H%M)"
git checkout -b "$BRANCH"

# 2. (skill writes files to docs/)

# 3. Stage + commit
git add docs/
git diff --cached --quiet && {
  echo "No changes to commit. Skipping PR."
  exit 0
}
git -c user.name="business-docs plugin" -c user.email="business-docs@local" \
  commit -m "docs: <one-line summary> (auto-generated)"

# 4. Push
git push -u origin "$BRANCH"

# 5. Open PR — prefer gh CLI
if gh auth status >/dev/null 2>&1; then
  gh pr create \
    --title "docs: <one-line summary>" \
    --body "$(cat docs/_AUTOGENERATED.md 2>/dev/null || echo 'Auto-generated by business-docs plugin.')" \
    --base "$(gh repo view --json defaultBranchRef -q .defaultBranchRef.name)" \
    --head "$BRANCH"
else
  # Fallback: REST API with GITHUB_TOKEN
  REPO_OWNER_NAME="$(git remote get-url origin | sed -E 's#.*github.com[:/]([^/]+/[^/.]+)(\.git)?#\1#')"
  DEFAULT_BRANCH="$(curl -s -H "Authorization: Bearer $GITHUB_TOKEN" \
      "https://api.github.com/repos/$REPO_OWNER_NAME" | jq -r .default_branch)"
  curl -s -X POST \
    -H "Authorization: Bearer $GITHUB_TOKEN" \
    -H "Accept: application/vnd.github+json" \
    "https://api.github.com/repos/$REPO_OWNER_NAME/pulls" \
    -d "$(jq -n --arg title "docs: <one-line summary>" \
                --arg head "$BRANCH" \
                --arg base "$DEFAULT_BRANCH" \
                --arg body "$(cat docs/_AUTOGENERATED.md 2>/dev/null || echo 'Auto-generated by business-docs plugin.')" \
                '{title:$title, head:$head, base:$base, body:$body}')"
fi
```

## PR title and body conventions

**Title** — one line, prefixed `docs:`, summarizes the skill run:
- `docs: initial business documentation (auto-generated)`
- `docs: refresh — 3 modules, 5 connection updates`
- `docs: regenerate billing module`
- `docs: derive checkout workflow from code`
- `docs: regenerate inter-module connection map`

**Body** — should include:
1. A one-paragraph summary of what was generated.
2. Counts (modules, connections, BR-AUTO entries, pain items).
3. The contents of `docs/_AUTOGENERATED.md` (the review checklist) if present.
4. A footer: `🤖 Generated by the business-docs plugin. Review the {{TBD}} fields in docs/_AUTOGENERATED.md before merging.`

## Cross-fork handling

If the user clones a fork (origin = fork, upstream = original repo):
- Default: open the PR against `upstream` (the original repo) using the user's fork as the head.
- Detect by: `git remote get-url upstream` succeeds → use `upstream`'s default branch as base.
- If only `origin` exists, open PR against `origin`'s default branch.

## Failure modes

| Situation | Behavior |
|---|---|
| No credentials at all | Skip push/PR. Print: "Docs written to `<path>`. To open a PR: set `GITHUB_TOKEN` or run `gh auth login`, then `cd <path> && git push -u origin <branch> && gh pr create`." |
| Push rejected (no write access) | Print: "You don't have write access to `<repo>`. Fork it and re-run, or push the branch to your own remote manually." |
| `git diff --cached --quiet` (no changes) | Skip the commit and PR. Tell the user the docs are already up-to-date. |
| PR already exists for this branch | `gh pr create` will fail. Detect, then `gh pr edit --body ...` to refresh it instead. |

## Don't

- **Never push directly to the default branch.** Always a feature branch.
- **Never force-push.** If a follow-up run conflicts, create a new branch with `-2`, `-3` suffix.
- **Never include credentials in commits or PR bodies.** If the clone URL was tokenized, scrub it from `git remote -v` before referencing.
